# MVP 本番確認手順

対象環境: `https://mtaskit.vercel.app`

## 1. 事前準備

- Google ログイン済みであることを確認する
- `/settings` のテストデータ生成機能で任意の既存プロジェクトに数件のタスクを作成しておく
- タグが未作成なら 2 件以上作成しておく

## 2. 親子プロジェクト

1. 任意の通常プロジェクトを開く
2. 右ペインまたはヘッダのプロジェクト編集領域で `New subproject` に名前を入力する
3. `Add subproject` を押す
4. 左サイドバーの Projects で、親の下にインデント付きで子プロジェクトが表示されることを確認する
5. 子プロジェクトを開く
6. 親プロジェクト変更ドロップダウンに `Inbox` と `完了` が表示されないことを確認する
7. 親プロジェクト変更ドロップダウンに自分自身が表示されないことを確認する

期待結果:
- サブプロジェクトが作成される
- 親子関係が一覧で分かる
- 不正な親候補が出ない

## 3. 親プロジェクト集約表示

1. 親プロジェクトを開く
2. 子プロジェクトで 1 件タスクを作成する
3. 親プロジェクトへ戻る
4. `子プロジェクトを含める` が ON の状態で、子プロジェクトのタスクが一覧に出ることを確認する
5. `子プロジェクトを含める` を OFF にする
6. 子プロジェクトのタスクが一覧から消えることを確認する

期待結果:
- ON で子孫タスクを含む
- OFF で親自身のタスクだけになる

## 4. タスク移動

1. 任意のプロジェクト画面でタスクを選択する
2. 右ペインの編集フォームでプロジェクトドロップダウンを変更する
3. `Save task` を押す
4. 移動先プロジェクトを開く
5. タスクが移動先に存在することを確認する

期待結果:
- タスクが選択したプロジェクトへ移動する
- 元プロジェクトの一覧からは消える

## 5. 完了と再オープン

1. 任意の未完了タスクで `Complete` を押す
2. 左サイドバーから `完了` プロジェクトを開く
3. 完了したタスクが表示されることを確認する
4. `Reopen` を押す
5. `Inbox` を開く
6. 再オープンしたタスクが `Inbox` に戻っていることを確認する

期待結果:
- 完了時は `完了` プロジェクトへ移動する
- Reopen 時は `Inbox` に戻る

## 6. タグと検索

1. タスクにタグを付けて保存する
2. 検索欄へタスク名の一部を入力して `Search`
3. 目的のタスクが検索結果に出ることを確認する
4. タグ削除後も他タスクの完了や編集が失敗しないことを確認する

期待結果:
- タイトル検索が動く
- タグ削除後に `Tag not found` が残らない

## 7. 保存ビュー

1. 任意の条件で保存ビューを作成する
2. `/view/[viewId]` を開く
3. 条件に応じた一覧が表示されることを確認する
4. `Include child projects` を ON にして親プロジェクトを条件に含める
5. 子プロジェクトのタスクも含まれることを確認する

期待結果:
- 保存ビューが作成できる
- ビュー条件が反映される
- 子孫プロジェクト条件も有効

## 8. 競合 UI

1. 同じタスク編集画面を 2 タブで開く
2. 片方のタブでタイトルを変更して保存する
3. もう片方のタブで別の変更を保存する
4. 対象別の競合メッセージが表示されることを確認する
5. `Reload latest data` を押す
6. 最新状態が再取得されることを確認する

期待結果:
- `task / project / view / tag` ごとの競合文言が表示される
- `再読み込みすると未保存の変更は失われます。` が表示される
- 再読み込み後に最新状態へ戻る

## 9. 完了条件

- Google ログインが安定して動く
- 親子プロジェクトが作成・表示・変更できる
- 子プロジェクト含有トグルが動く
- タスク移動が動く
- 完了と Reopen が期待通り動く
- 検索と保存ビューが使える
- 競合メッセージと再読み込み導線が使える
